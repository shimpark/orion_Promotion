컴포넌트 페이지의 사용 의도와 활용 방법

작은 규모의 프로젝트 작업시 컴포넌트 페이지의 활용도는 높지않으나
규모가 크고 디자인 요소의 공통부분이 많을 경우 프로젝트 초반 공통 컴포넌트를 선 개발 후 페이지 작업을 하면
작업 속도 향상 및 코드 유지보수가 쉬워지며 다른 작업자의 백업 또는 협업시 코드 재활용성이 높아진다.

버튼

- 기본

  • 미리 정의된 버튼 스타일이 포함되어 있으며 각각의 의미와 목적에 맞게 사용하면 됩니다.
  • 색상은 config.scss에 정의해서 사용하도록 권장합니다.

- 사이즈별 버튼

  • 버튼의 공통 크기를 지정하여 사용할 수 있습니다.
  • 사이즈: small / medium / large / xlarge

- 유형별 버튼

  • 테두리 또는 라운드 버튼을 지정 할수 있습니다.

- 버튼 input

  • 버튼이 함께 있는 input 입니다.
  • 버튼의 사이즈에 따라 small / medium / large 로 구분할 수 있습니다.

- 달력 input

  • 달력이 포함되어있는 input 입니다.
  • 달력은 년 / 월 / 시간 달력으로 구분되어 있습니다.
  • .month클래스를 추가하면 월 달력이 됩니다.
  • .time클래스를 추가하면 시간 달력이 됩니다.






- 파일업로드 input

  • 파일을 업로드 할수 있는 input 입니다.
  • 파일명을 바꾸는 스크립트가 페이지내에 첨부되어야 합니다.(개발 추가작업을 위함)
※ 안내문구입니다.

- textarea

  • textarea 입니다.
  • 사이즈 - full / sm / md / lg
  • data-maxlength 속성에 글자수를 넣어주시면 됩니다.



셀렉트박스(드롭다운)

- 일반

  • 일반적인 select박스 입니다.
  • 사이즈 : small / medium / large



- 셀렉트릭(라이브러리)

  • 라이브러리를 이용한 select박스(드롭다운메뉴) 입니다.
  • 사이즈 : small / medium / large / float

  • 비동기후 셀렉트릭 안될때(업데이트) 함수 - window.refreshSelectric()

체크박스/라디오버튼

- 체크박스

  • 체크박스 유형은 round / square 두가지가 있습니다.

- 라디오버튼

    - 가로형

    • .horizontal 클래스를 추가하면 가로형이 됩니다.

    - 라벨없음

    • 라벨이 없는 경우 .solo 클래스를 추가하면 됩니다.
    • 라벨이 없어도 label 태그는 존재해야 합니다.

    아코디언

    • .accordion-container클래스에 .solo 클래스를 붙히면 리스트의 토글이 각각 적용됩니다.
    • 아코디언을 펼친상태에서 렌더링 하려면 .accordion-list클래스에 .active 클래스를 추가하고 .accordion-body태그에 style="display:block"속성을 넣어줘야 합니다.
    • data-speed 속성을 이용해 토글 속도를 조절할수 있습니다.

    pagination

    • 리스트의 공통 페이징입니다.
    • 이전이나 다음페이지가 없을경우.controller 클래스에 .disabled 클래스를 붙히면 버튼이 비활성화 됩니다.

    슬라이드(swiper.js)

    • 비동기 후 슬라이드 안될때(업데이트) 함수 - window.sliderUpdate()
    • .swiper-container클래스에 슬라이드별로 클래스를 추가해 사용해주세요.
    • 슬라이드 스크립트 및 옵션은 common-ui.js를 참고해주세요.
